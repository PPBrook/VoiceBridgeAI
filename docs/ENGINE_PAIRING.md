# 句中 / 句末不同模型 — 配置教程

VoiceBridgeAI 把翻译拆成两层：

| 层 | 何时调用 | 目标 |
|---|---|---|
| **句中（partial）** | 说话过程中，频繁更新 | 低延迟草稿，让用户先看到中文 |
| **句末（final）** | 一句结束、定稿时 | 更自然的中文，可结合草稿润色 |

**推荐做法：** 句中用「快、便宜」的机器翻译或 LLM 快译；句末用「准、自然」的 LLM 润色。两层选**不同接口**通常效果更好。

---

## 一、先理解两类接口

### 机器翻译（MT）

腾讯 TMT、百度、Google、DeepL、Argos 离线等。

- 适合句中：响应快、按次计费低
- 一般不允许句中、句末选**同一家 MT**（避免定稿层无额外价值）
- **例外：Argos** 可在句中、句末各选一次（离线场景常用）

### 大模型（LLM）

七牛 AI、阿里云 DashScope、DeepSeek、OpenAI 等。

- **句中**：`polish=False`，短 prompt 快译
- **句末**：`polish=True`，参考句中草稿做润色
- **允许句中、句末选同一家 LLM**（模式不同，不是重复调用）

### 不翻译（none）

句末选 **不翻译（沿用句中）** 时，定稿直接使用句中草稿，不再调用额外接口。适合只想快速看字幕、或离线只有 Argos 且不想重复润色的场景。

---

## 二、系统组合规则

控制台会自动限制无效组合：

| 情况 | 是否允许 | 说明 |
|---|---|---|
| 句中 TMT + 句末 TMT | ❌ | 句末下拉不会出现 TMT |
| 句中 Argos + 句末 Argos | ✅ | 离线默认可用；句末仍走 Argos 润色路径 |
| 句中 Argos + 句末 **不翻译** | ✅ | 定稿沿用句中草稿，零额外延迟 |
| 句中七牛 + 句末七牛 | ✅ | 快译 + 润色，同一 API Key |
| 句中 TMT + 句末七牛 | ✅ | **最常用国内组合** |

下拉框会按 **句中推荐 MT / 句末推荐 LLM** 分组，便于选择。

## 三、控制台配置步骤

### 1. 启动服务

```bash
cp .env.example .env   # 按需填写密钥
./run.sh
```

打开 [http://127.0.0.1:8765](http://127.0.0.1:8765)（端口见 `VOICEBRIDGE_PORT`）。  
启动后会**自动测试** `.env` 里已配置的接口（`AUTO_TEST_ON_START=0` 可关闭）。

### 2. 接口配置（填 Key → 保存 → 测试）

打开 **[http://127.0.0.1:8765/config](http://127.0.0.1:8765/config)**：

1. 展开需要的厂商，填写密钥
2. 点 **保存配置**（合并写入项目根目录 `.env`）
3. 对每个要用到的能力点 **测试**（识别 / 句中 / 句末），或点 **一键测试全部**

只有**测试通过**的云端接口才会出现在 **引擎设置** 下拉框。  
离线默认可用项（本地 Whisper、Argos、Google 等）**无需**测试。

`VERIFIED_*` 标志只在**当前进程**内生效，不会写入 `.env`；重启后靠启动自动测试或手动 **一键测试全部** 恢复。

**示例：想用「腾讯识别 + TMT 句中 + 七牛句末」**

| 厂商 | 需测试的能力 |
|---|---|
| 本地 Whisper | 识别（可选，默认可用） |
| 腾讯云 | 识别、句中（TMT 与 ASR 共用 Secret） |
| 七牛 AI | 句中、句末（各测一次） |

**OpenAI 注意：** 识别测试只验证 Key 是否有效；句中/句末走 Chat Completions，若返回 429 通常是账户配额不足，而非 Key 错误。

### 3. 引擎设置（三层组合）

在首页 **[http://127.0.0.1:8765/](http://127.0.0.1:8765/)** 的 **引擎设置**：

| 字段 | 推荐 |
|---|---|
| **语音识别** | 国内低延迟 → 腾讯云；离线 → 本地 Whisper |
| **句中翻译** | 国内 → 腾讯 TMT；要 LLM 快译 → 七牛 / DeepSeek |
| **句末润色** | 国内 → 七牛 / 阿里云 / DeepSeek LLM；或 **不翻译** |
| **纠正模式** | 一般选「标准纠正」 |

选完后会自动保存到内存并写入 `.env`。面板下方会提示当前组合是否合理。

### 4. 开始捕获

点 **捕获音频**，在 Chrome 中选择**带英文音频的标签页**，并勾选「分享标签页音频」。

## 四、推荐组合方案

### 方案 A：国内低延迟（推荐）

适合网课、技术分享、国内网络。

| 层 | 接口 | 需配置 |
|---|---|---|
| 识别 | 腾讯云 | `TENCENT_ASR_*` |
| 句中 | 腾讯 TMT | 同上 + `TMT_REGION` |
| 句末 | 七牛 AI | `QINIU_AI_API_KEY` |

`.env` 片段：

```env
ASR_PROVIDER=tencent
PARTIAL_PROVIDER=tmt
FINAL_PROVIDER=qiniu

TENCENT_ASR_APP_ID=你的AppId
TENCENT_ASR_SECRET_ID=你的SecretId
TENCENT_ASR_SECRET_KEY=你的SecretKey
TMT_REGION=ap-shanghai

QINIU_AI_API_KEY=你的Key
QINIU_AI_MODEL=qwen-turbo
```

### 方案 B：国内全 LLM

句中、句末都用同一家，靠模式区分。

| 层 | 接口 |
|---|---|
| 识别 | 腾讯云 或 OpenAI |
| 句中 | DeepSeek |
| 句末 | DeepSeek（控制台允许） |

适合已有 DeepSeek Key、对延迟要求不极端的场景。

### 方案 C：全本地 / 离线（默认）

无需任何云端 Key，开箱即用。

| 层 | 接口 |
|---|---|
| 识别 | 本地 Whisper |
| 句中 | Argos 离线 |
| 句末 | Argos，或 **不翻译**（沿用句中草稿） |

`.env` 片段：

```env
ASR_PROVIDER=local
PARTIAL_PROVIDER=argos
FINAL_PROVIDER=argos
WHISPER_MODEL=base.en
REVISE_MODE=balanced
```

首次运行 Whisper / Argos 会下载模型，稍等即可。

### 方案 D：海外网络

| 层 | 接口 |
|---|---|
| 识别 | OpenAI Whisper |
| 句中 | Google 或 DeepL |
| 句末 | OpenAI LLM |

需能访问对应海外服务；Google 句中/句末无需 Key，但网络要通。

## 五、Chrome 扩展

Web 控制台配好后：

1. `chrome://extensions` → 加载 `extension/`
2. 扩展弹窗中的下拉与控制台一致（云端项需已测试通过）
3. 弹窗保存引擎时会同步 `POST /api/engine/settings` 到本地服务
4. 句中 / 句末同样遵循「MT 不可重复（Argos 除外）、LLM 可同一家、可选不翻译」规则

默认与服务端一致：`local + argos + argos`。

## 六、常见问题

**Q：重启后云端下拉变空了？**  
A：`VERIFIED_*` 只在当前进程有效。重启 `./run.sh` 后会自动再测；也可打开 `/config` 手动 **一键测试全部**。

**Q：句末下拉里没有我想选的？**  
A：可能和句中选了同一 MT（被过滤，Argos 除外），或该接口句末能力未测试通过。回 `/config` 点对应 **句末 · 测试**。

**Q：句中七牛 + 句末七牛算重复吗？**  
A：不算。句中是快译，句末是润色，共用 Key、不同 prompt。

**Q：只想快速看字幕，不要句末润色？**  
A：句末选 **不翻译（沿用句中）**；或选 MT 句中 + 不同 LLM 句末。

**Q：OpenAI 识别通过了，句中/句末 429？**  
A：识别测试只验 Key；Chat 接口需账户有余额或开通计费。

**Q：如何关闭启动自动测试？**  
A：`.env` 中设 `AUTO_TEST_ON_START=0`。

## 七、数据流示意

```
英文音频
  → 识别（ASR）→ 英文句
       → 句中翻译（partial，快）→ 中文草稿字幕（边说边更新）
       → 句末润色（final，准）→ 定稿中文字幕（一句结束时替换；选「不翻译」则沿用草稿）
```

两层模型不同，才能在**延迟**和**质量**之间取得平衡。
